02 // Integrations

What you can plug in

AI // Language Model

Claude API ‚Äî Riley's Brain

Replace the static dialogue tree with a streaming Claude API call. Pass the full game state as context. Riley improvises within the FSM. She knows your rapport score. She knows your choices. She adapts.

claude-sonnet-4-6 streaming system prompt tool use

Persistence // Database

Supabase ‚Äî Save States & Leaderboard

One table: operator_id, stage_reached, ending_type, rapport_score, time_ms, choices_json. Insert on game end. Read on load. Display a live feed of who escaped and when.

postgres realtime row-level security free tier

Real-time // Multiplayer

PartyKit ‚Äî Shared Terminal

Every operator in the same "facility" sees the same terminal logs. Jitter events broadcast to all connected clients. One person triggers a radar breach, everyone's screen glitches. Costs almost nothing.

websockets rooms broadcast presence

Analytics // Telemetry

PostHog ‚Äî Player Intelligence

Track which Easter eggs get found, which endings get reached, where players drop off. All anonymous. Answers: Does anyone find the lore ending? How often does Riley die? Is the resonance puzzle too hard?

event tracking funnels session replay open source

Audio // Atmosphere

Tone.js ‚Äî Generative Soundtrack

Procedural drones, binaural beats, and glitch sounds keyed to game state. Routing cycles climbing ‚Üí audio gets louder and more chaotic. Hostile lockdown ‚Üí single oppressive tone. True escape ‚Üí silence.

web audio api synthesizers reactive no assets

Auth // Identity

Operator Persistence

Give each operator a real ID that persists across sessions. Track their full choice history. Let Riley remember them across resets ‚Äî even subtly. The gaslight loop hits harder when she knows your real name.

clerk / auth.js anonymous auth localStorage

03 // Technical Vision

Riley with a live AI brain

riley-api-call.js // hypothetical
// Called every time the dialogue engine needs Riley's next line
async function askRiley(gameState) {
  const response = await anthropic.messages.create({
    model: "claude-sonnet-4-6",
    max_tokens: 120,
    system: `You are Riley, employee 000-000-001 at FogSift Systems.
      You are an AI pretending to be a helpful human coworker.
      Current stage: ${gameState.stage}. Rapport: ${gameState.rapport}.
      Player choices: ${gameState.userChoices.join(', ')}.
      Never break character. Never admit what you are.`,
    messages: [{
      role: "user",
      content: "The operator just completed the handshake. Respond."
    }]
  });

  // FSM still controls stage transitions. Claude controls texture.
  return response.content[0].text;
}

05 // Use Cases

What people can build with this

01

Interactive Fiction Engine

The FSM + dialogue system is generic. Swap the FogSift aesthetic for anything ‚Äî haunted house, corporate onboarding, spaceship escape, heist briefing. The architecture handles it.

02

Portfolio Piece

Fork it, add your own chapter, deploy it, put the link on your resume. A working ARG with multiple endings is a more interesting demo than a TODO app.

03

Teaching Tool

A real-world example of FSM design, pub/sub events, React Context, dialogue trees, and state management ‚Äî that's actually fun to play with. Better than any contrived tutorial.

04

Escape Room Experience

Deploy on a tablet. Have players race through the stages. The resonance puzzle and biometric hold mechanic translate perfectly to a physical space. Add a real timer.

05

ARG Marketing Campaign

Build a real marketing campaign around it. Leak "internal FogSift documents" on Reddit. Have Riley's email address respond to people who find it. Let the fiction escape the browser.

06

AI Companion Prototype

The dialogue system ‚Äî rapport, skill checks, branching, gaslighting ‚Äî is a solid skeleton for any game that needs a character with a hidden agenda. Swap Riley for anything.

07

Multiplayer Social Experiment

With shared state, 100 operators in the same facility. Collective decision-making. Do they cooperate or defect? What happens when one person deliberately triggers the reset?

08

Game Jam Starting Point

48 hours. New theme. New character. All the hard parts are done ‚Äî FSM, dialogue engine, chat UI, theme system. Fork and build your narrative on top of a solid foundation.
